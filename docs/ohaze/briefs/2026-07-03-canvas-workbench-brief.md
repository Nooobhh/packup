# 画布工作台 (canvas-workbench) — Feature Brief
> 给 haze 看。spec 在 docs/ohaze/specs/2026-07-03-canvas-workbench-design.md 给 Codex 看,你不用看。

## 这是干嘛的
把行程页从「生成结果展示页」升级为「可持续编辑的画布工作台」:笔记解析出的地点变成卡片,自由编排进每天的行程,智能排程之后仍可随手调整——对标圆周旅迹已验证的编辑闭环,但保持 packup 单机零费架构。

## 给谁用
haze 本人(单机版):导入小红书笔记规划真实旅行;或不带笔记、从零手搭行程的场景。

## 用户场景 (Scenarios)

### Scenario 1: 笔记导入后自由编排(happy path)
- Given 已导入小红书链接,在选点页勾选部分地点生成了行程
- When 进入工作台,把待计划池里一张落选卡片拖进 Day 2,再调换 Day 1 两张卡片顺序
- Then 卡片落位、相邻交通段自动重算(只算受影响段)、地图路线同步刷新;刷新页面编排保留

### Scenario 2: 手动从零搭行程(无笔记)
- Given 不想导入笔记
- When 首页选「手动从零」,填目的地+天数进入空工作台,搜索「灵隐寺」加入 Day 1
- Then 空行程建立、搜索结果入天成卡、交通段与地图正常工作,后续体验与导入流程一致

### Scenario 3: 智能排程后再调(AI + 人协作)
- Given Day 3 有 5 张顺序凌乱的卡片
- When 点该天「智能排程」,然后手动把晚餐卡拖到末位
- Then 一键后顺序按地理就近优化;手动微调不被覆盖,交通段随之更新

### Scenario 4: 失败与冲突恢复
- Given 正在编辑行程
- When 某段交通计算失败(高德超时),或另一窗口先改了行程
- Then 失败段单独显示「点击重试」、不炸整个操作;并发冲突弹「行程已更新」并自动刷新(409 语义)

## "完成的样子" Checklist
- [ ] 能:池卡拖入任意天、天内排序、跨天移动,刷新不丢
- [ ] 能:增删天、改天主题;删天时卡片回池不消失
- [ ] 能:编辑卡片备注/开始时间/时长
- [ ] 能:交通段四方式切换(步行/骑行/驾车/公交)+ 全局「长短途」两档默认偏好
- [ ] 能:搜索高德 POI 加入池或某天
- [ ] 能:点卡片看详情——地址/营业时间/评分/推荐理由/来源笔记引用(可跳原笔记)
- [ ] 能:地图按天着色、总览/单天切换、卡片↔地图双向联动
- [ ] 失败时能:段级重试;409 冲突自动刷新不覆盖
- [ ] 不能:池↔天之间卡片丢失或重复(守恒)

## 不做什么 (Out of Scope)
- 导出长图 / 分享 / 协作 / 账号(全部后置)
- 卡片照片上传、Note/CheckList 手账区
- 无笔记纯智能生成(城市+偏好直接出行程)
- 移动端体验优化(桌面浏览器优先)
- 视觉美化(功能优先,页面视觉 haze 后续重做)

## Scope 决策
- **模式**: Reduction
- **理由**: 从「对齐竞品全功能」收敛到「单机编辑闭环」,首版可测试可交付;协作/导出等画布稳定后跟进。

## Claude 替你决定的关键技术方向 (Phase 1.5 后回填,事后扫一眼)
- 拖拽用 @dnd-kit/core + @dnd-kit/sortable 列表模型(泳道看板式),不引入自由画布库(react-flow)
- 「智能排程」复用现有兜底重排的最近邻算法(plan.ts nearestClusterOrder),本地计算不走 LLM:秒回、零费、结果可测
- 待计划池持久化为 40-plan.json 内新 pool 字段,与 days 同文件同并发防线;仅「已验证有坐标」的落选候选入池,未验证仍进过滤区
- 骑行走高德 v4 bicycling 端点(现有 key 免费额度内),v4 响应结构单独适配,空结果降级直线估算
- 全局交通偏好 = 改造现有段推荐函数的距离档规则;改偏好后提供「按新偏好重算」显式操作,不静默全量重算
- POI 搜索复用高德 place/text 多结果模式、不逐条查详情(省配额);营业时间/评分缺省显示未知
- 编辑走乐观 UI:先本地变更后发 PATCH(返回全量 plan 对齐状态),失败回滚,409 自动刷新
- 手动从零:放宽 TripInput.links 允许空数组,/api/generate 保持链接非空校验(导入语义不变)
- 重新选点=重建行程,会覆盖工作台编辑(沿用既有重排语义),选点页加文案提示
