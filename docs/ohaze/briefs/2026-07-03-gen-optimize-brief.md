# packup 0.3.0 生成优化 — Feature Brief
> 给 haze 看。spec 在 docs/ohaze/specs/... 给 Codex 看,你不用看。

## 这是干嘛的
把 packup 从「一次性生成器」升级为「生成 + 轻确认的规划工作台」:一句话自然语言输入,两段式生成(先选点、后排程),时段制行程 + 真实路线,机器耗时 3-5 分钟出结果。

## 给谁用
haze 自用(本机部署)。触发者 = 行前规划的旅行者本人。

## 用户场景 (Scenarios)

### Scenario 1: 自然语言 + 链接一次生成(happy path)
- Given 我复制了 3 条小红书香港攻略链接
- When 我在搜索框输入「香港3天2晚 city walk+美食」、粘贴链接、点生成
- Then 几分钟内进入**选点确认页**:提取出的 POI 按验证状态列出,已验证默认勾选、未验证默认不勾但可勾
- When 我取消两个不想去的,点「排程」
- Then 约 1 分钟后看到行程页:3 天 × 上午/下午/晚上时段,每个节点带推荐交通方式,地图画真实路线

### Scenario 2: 重排与改交通(边界)
- Given 行程页已生成
- When 我通过重排 API 把 Day 1 两个节点换序,或把某段交通改为步行
- Then 只有受影响的相邻段被重算(按需最少次数高德调用——改单段 1 次、相邻互换 3 次、顺序不变 0 次,不重跑 LLM),路线图同步更新

### Scenario 3: 全部 POI 查证失败(失败/恢复)
- Given 我贴的笔记 POI 全部高德查证失败
- When 排程完成
- Then 行程**不为空**:未验证 POI 保留、带「未验证」标注排入时段;失败链接处显示一句话摘要而非 claude 命令原文

场景内细节口径:
- query 解析:目的地必出;天数缺省时走已有的「按内容推荐天数」;偏好文本直接进 plan prompt
- 邻近聚合:相邻 ≤250m 的 POI 在时间轴合并为组合节点(时长为成员之和)、地图单 marker + 成员说明

## "完成的样子" Checklist
- [ ] 能用「杭州3天旅游攻略」「吉隆坡5天city walk+美食」这类一句话 + 链接生成行程;表单只剩搜索框 + 链接粘贴区
- [ ] 能在选点页勾选/取消 POI,只有选中的进入排程
- [ ] 行程按上午/下午/晚上时段展示,不再输出分钟级时刻
- [ ] 每段交通是按距离推荐的 mode;地图画高德真实 polyline(公交分段拼接,缺段降级直线)
- [ ] 机器耗时合计 ≤5 分钟、目标 3 分钟(提交→选点页 + 排程→行程页,不含选点停留)
- [ ] 重排/改交通 API 只重算受影响段,不重跑 LLM
- [ ] POI 全部未验证时行程不为空
- [ ] failReason 为一句话摘要
- [ ] 旧 trip 落盘数据仍能打开(契约向后兼容)

## 不做什么 (Out of Scope)
- 无链接生成模式(已拍板延后)
- 拖拽/编辑交互壳与视觉美化(本期 checkbox + API 先行)
- 自动搜小红书(Backlog,待 spike)
- 协作、分享、自建 POI 库(对标产品有,我们不做)

## Scope 决策
- **模式**: Hold Scope
- **理由**: 范围已经过两轮对齐收敛,6 项互相支撑构成完整工作流(输入→选点→快速排程→真实路线),再加威胁 3-5 分钟工期、再减破坏工作流完整性。

## Claude 替你决定的关键技术方向 (Phase 1.5 后回填,事后扫一眼)
- query 解析走「正则规则先行、LLM 一次兜底」,识别不出目的地直接 400 提示,不猜
- 两段式不加新管线段:generate 跑到 ground 停(`toStage`),选点落 `25-selection.json`,plan 段读它过滤;旧 trip 无此文件走全量(兼容)
- plan 段 LLM 只做「分天分时段」且输出 poiId 引用(根治按名字匹配导致的空行程 bug),顺序由最近邻算法排,LLM 修复循环整个删除
- POI 时长不再让 LLM 排:按笔记 suggestedDuration 解析,缺失按类型给默认值(景点90/餐饮60分钟等)
- 交通推荐规则:<800m 步行,其余公共交通,公交超 90 分钟时自动比价驾车
- polyline 从高德 direction 现有响应解析(零新增 API 调用),超 500 点抽稀,解析失败降级直线
- 聚合用并查集(250m 阈值),组内成员间不调路线 API(直线+步行估算)
- 重排 API 语义:cluster 作为整体节点移动;重复提交选点 = 覆盖并强制重跑 plan
- 保险不变式:选中 POI 非空 ⇒ 行程非空(LLM 分天失败时按地理就近确定性均分),单测断言
- 时间预算集中成常量表(分天 LLM 90 秒、提取单笔记 120 秒等),段级到点「部分成功」继续而非炸管线;预算总和 = 300 秒整,用单测锁死,防未来悄悄调大
- 分天 LLM 不重试:失败直接走地理就近兜底(省失败路径 90 秒等待),行程页带提示可手动重跑 plan 段
- 重排只重算「新顺序中新出现的相邻对」:改单段交通 1 次调用、相邻互换 3 次(物理下界),顺序不变 0 次
